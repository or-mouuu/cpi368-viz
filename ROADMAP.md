# 功能構想備忘（Roadmap）

> 記錄日期：2026-07-11。已上線的第一版：sparkline 卡片格 + 四類型/七大類篩選 + detail overlay。
> 以下是討論過、尚未實作的方向，依優先序整理。

## 高優先：資料加值（已確認有興趣）

### 1. 平均零售價格（絕對價格）
- 主計總處「重要項目平均價格」資料集：雞蛋每公斤幾元、白米每公斤幾元等**真實價格**。
- 只涵蓋幾十個民生重要品項，但體感最強：「2013 年一公斤雞蛋 65 元 → 現在 95 元」遠比指數有感。
- 用法：detail overlay 對有價格資料的品項多顯示一條「真實價格」曲線或前後對照。
- 待辦：查證資料集實際格式、涵蓋品項數、與 368 項目的對映方式。

### 2. 工資換算購買力（「買這個要工作幾分鐘」）
- 基本工資（或平均薪資）歷年序列，資料量極小。
- 殺手級視角：物價漲 10% 但同期基本工資漲 40% → 用勞動時間換算很多東西其實變便宜。
- 直接回應「薪水沒漲、物價一直漲」的最大迷思，最有分享傳播力的一張圖。
- 待辦：確認用基本工資還是受僱員工經常性薪資（中位數/平均），兩者敘事不同。

### 3. 國際原物料對照（專題式）
- World Bank / FRED 免費資料：國際小麥、原油、黃小玉價格。
- 對照麵包、麵條、汽油：「全球小麥漲完多久台灣麵包才跟漲？漲上去有跌回來嗎？」（rocket and feather 效應）。
- 適合做成一兩篇專題版面，不是通用功能。放最後做。

## 中優先：既有資料就能做

### 4. 年增率序列（資料已在手上，pipeline 目前丟掉）
- XML 內同時有「原始值」與「年增率」，保留後可區分「十年累積漲最多」（存量）vs「現在正在漲」（動能）。
- **股市化的理解框架**（使用者覺得有趣）：飆漲中／早就漲完了／最近才起漲／牛皮盤整／長期下跌——同一條線的兩個維度，很像看個股的長線走勢＋近期動能。
- 用法：「本月漲價進行式」榜單、detail 圖切換兩種視角、或做成分類徽章。

### 5. 權數表（treemap 與影響力榜）
- 每項目在 CPI 的權數（千分比），主計總處有公開。
- 解鎖：treemap（面積=權數、顏色=漲幅，一眼看出「痛在哪」）、「錢包影響力」榜（漲幅×權重）、個人通膨計算機的準確版。

### 6. 排序榜單 + 疊圖比較 + 搜尋
- 十年漲幅 TOP10／跌幅 TOP10 一鍵切換；品項搜尋框。
- 疊圖比較：選 2–4 個品項 normalize 到起點=100 疊在同一張圖，選擇寫進 URL query 可分享。

### 7. 走勢最像的品項
- pipeline 預算相關性（每項只留 top 5），detail overlay 顯示「和它走勢最像的品項」。
- 意外發現來源：例如麵包、洗剪髮、家事服務費同受工資推動。

## 敘事模組（編輯成本高、記憶點最強）

### 8. 物價迷思驗證區
- 問答互動：先拋直覺（「你覺得呢？」）→ 翻開數據答案。
- 候選題：「萬物齊漲？」（其實 30 項十年在跌）、「外食漲最兇？」、「學費一直漲？」（近乎凍漲）、「菜價都是颱風害的？」。
- 前置作業：先跑一輪資料探勘挑出 4–6 個答案反直覺的題目。

### 9. 事件標註
- 在特定品項圖上標事件：香菸 2017 菸稅+20 元、雞蛋 2023 蛋荒、2022 全球通膨、歷年基本工資調漲點。
- 工程量小，主要是查證與編輯。

## 待重新思考：四類型分類定義

- 現況問題（實際數據）：「漲相很兇」（>15%）佔 161/358 ≈ 45%，快一半的品項都「很兇」，標籤失去鑑別力；且「隨季節」優先蓋過漲跌——季節波動大**且**十年漲四成的蔬果會被歸成「只是季節性」，隱藏了它的通膨。
- 方向：漲幅維度與波動維度應該正交——「隨季節」改成可疊加的**徽章/標記**而非互斥分類；「很兇」門檻重新校準（例如取前 20–25% 分位數）讓它真的稀有。
- 若採股市框架，可與年增率動能維度合併設計，但要守住「操作直覺」原則：主分類一個維度，其他都是徽章。

## 設計原則（使用者定調）
- 整體操作以直覺為主，不堆功能；每加一個維度都用「徽章/標籤」方式疊加，不增加主分類複雜度。
