# 功能構想備忘（Roadmap）

> 記錄日期：2026-07-11。已上線的第一版：sparkline 卡片格 + 四類型/七大類篩選 + detail overlay。
> 以下是討論過、尚未實作的方向，依優先序整理。

## 高優先：資料加值（已確認有興趣）
<!-- 2026-07-11 更新：權數表、比較工具、品項關聯經使用者確認同列高優先評估 -->


### 1. 平均零售價格（絕對價格）
- 主計總處「重要項目平均價格」資料集：雞蛋每公斤幾元、白米每公斤幾元等**真實價格**。
- 只涵蓋幾十個民生重要品項，但體感最強：「2013 年一公斤雞蛋 65 元 → 現在 95 元」遠比指數有感。
- 用法：detail overlay 對有價格資料的品項多顯示一條「真實價格」曲線或前後對照。
- 待辦：查證資料集實際格式、涵蓋品項數、與 368 項目的對映方式。

### 2. 工資換算購買力（「買這個要工作幾分鐘」）
- 基本工資（或平均薪資）歷年序列，資料量極小。
- 殺手級視角：物價漲 10% 但同期基本工資漲 40% → 用勞動時間換算很多東西其實變便宜。
- 直接回應「薪水沒漲、物價一直漲」的最大迷思，最有分享傳播力的一張圖。
- 待辦：確認用基本工資還是受僱員工經常性薪資（中位數/平均），兩者敘事不同。

### 3. 國際原物料對照（專題式）
- World Bank / FRED 免費資料：國際小麥、原油、黃小玉價格。
- 對照麵包、麵條、汽油：「全球小麥漲完多久台灣麵包才跟漲？漲上去有跌回來嗎？」（rocket and feather 效應）。
- 適合做成一兩篇專題版面，不是通用功能。放最後做。

### 3.5 權數表、比較工具、品項關聯（使用者確認高優先評估，細節見下方 5/6/7）
- 權數表 → treemap、「錢包影響力」榜、個人通膨計算機。
- 疊圖比較 + 排序榜單 + 搜尋 → 自由探索的核心工具。
- 走勢最像的品項 → detail overlay 的意外發現來源。

## 中優先：既有資料就能做

### 4. 年增率序列（資料已在手上，pipeline 目前丟掉）
- XML 內同時有「原始值」與「年增率」，保留後可區分「十年累積漲最多」（存量）vs「現在正在漲」（動能）。
- **股市化的理解框架**（使用者覺得有趣）：飆漲中／早就漲完了／最近才起漲／牛皮盤整／長期下跌——同一條線的兩個維度，很像看個股的長線走勢＋近期動能。
- 用法：「本月漲價進行式」榜單、detail 圖切換兩種視角、或做成分類徽章。

### 5. 權數表（treemap 與影響力榜）
- 每項目在 CPI 的權數（千分比），主計總處有公開。
- 解鎖：treemap（面積=權數、顏色=漲幅，一眼看出「痛在哪」）、「錢包影響力」榜（漲幅×權重）、個人通膨計算機的準確版。

### 6. 排序榜單 + 疊圖比較 + 搜尋
- 十年漲幅 TOP10／跌幅 TOP10 一鍵切換；品項搜尋框。
- 疊圖比較：選 2–4 個品項 normalize 到起點=100 疊在同一張圖，選擇寫進 URL query 可分享。

### 7. 走勢最像的品項
- pipeline 預算相關性（每項只留 top 5），detail overlay 顯示「和它走勢最像的品項」。
- 意外發現來源：例如麵包、洗剪髮、家事服務費同受工資推動。

## 敘事模組（編輯成本高、記憶點最強）

### 8. 物價迷思驗證區
- 問答互動：先拋直覺（「你覺得呢？」）→ 翻開數據答案。
- 候選題：「萬物齊漲？」（其實 30 項十年在跌）、「外食漲最兇？」、「學費一直漲？」（近乎凍漲）、「菜價都是颱風害的？」。
- 前置作業：先跑一輪資料探勘挑出 4–6 個答案反直覺的題目。

### 9. 事件標註
- 在特定品項圖上標事件：香菸 2017 菸稅+20 元、雞蛋 2023 蛋荒、2022 全球通膨、歷年基本工資調漲點。
- 工程量小，主要是查證與編輯。

## 待重新思考：四類型分類定義 → 「走勢性格」方向（2026-07-11 更新）

- 現況問題（實際數據）：「漲相很兇」（>15%）佔 161/358 ≈ 45%，快一半的品項都「很兇」，標籤失去鑑別力；且「隨季節」優先蓋過漲跌——季節波動大**且**十年漲四成的蔬果會被歸成「只是季節性」，隱藏了它的通膨。
- 使用者顧慮：若主分類只剩「漲多少」三檔，網站會變成單純的漲幅排行榜，太無聊。
- **新方向：主分類改用「走勢性格」（walk-shape personas）**——「漲相」本來就是面相的雙關，分類「怎麼漲」比「漲多少」更貼題：
  - 「回不去了」：漲上去就站穩不回頭（例：鮭魚、外食）——峰值附近、回落小
  - 「炒一波就跌回」：暴漲後殺回來（例：蒜頭、口罩型商品）——峰值高但已回吐大半
  - 「緩緩爬」：十年線性慢漲（多數外食、服務類）
  - 「躺平／凍漲」：十年幾乎不動（例：學雜費、公共費率）
  - 「溜滑梯」：長期下跌（3C 通縮組）
  - 「雲霄飛車」：季節劇烈震盪（蔬果）
- 可量化的走勢指標（pipeline 可算）：十年總漲幅、**現價/歷史峰值比**（回不回得來）、峰值出現時點、回落幅度（drawdown）、季節變異數、近12個月年增率（動能）。
- 漲幅量級降級為卡片上的數字徽章（+X%），與性格分類並存：顏色回答「怎麼漲」、數字回答「漲多少」。
- 前置作業：對 358 項全部算走勢指標、看自然聚類，再從數據命名 4–6 個性格（不要憑空定義門檻）。
- **使用者定調（2026-07-11）**：「怎麼漲」正是「漲相」想表達的核心概念，確定往這個方向走；**先不套股市框架**，上面的性格名稱只是佔位，最終命名等分群結果出來再定。
- **分類規則 v3 已定案並實作上線（2026-07-11）**：宏觀三分（上漲／持平／下跌），上漲細分四種漲相＝持續上漲／漲後不跌／近期急漲／波動上漲；價格持平（含波動但趨勢平）；越來越俗（含波動但趨勢向下）。⚡事件徽章（6 項）＋〰️波動徽章。完整規則見 `docs/METHODOLOGY.md`。
- **待辦：網站「方法說明」獨立區塊**——把 METHODOLOGY.md 內容做成頁面區塊（資料來源、統計方法、分類規則）。
- **資料回溯 2000 年（不急）**：2021 基期序列僅到 2013；跨基期銜接因項目定義變動只能挑穩定品項做專題，見 METHODOLOGY.md 末節。

參考網站：https://thenewnormal.is/ （JS 渲染的敘事型資料視覺化站）。要參考的互動與動態：

1. **分段滑動（snap-scroll storytelling）**：前段敘事區（Hero、漲相說明等）滑一下切換一個畫面，一屏一個重點。
2. **雙模式切換**：切到「全部」列表 tab 時，退出 snap 模式、回到**自由滾動**的卡片格瀏覽——敘事模式與探索模式並存，切換要順。
3. **折線繪製動態**：每條折線圖進場時有 draw-on 動畫（stroke-dashoffset 逐漸畫出來的效果）。
4. **列表組裝動態**：多條折線／卡片從四面八方飛入、匯聚排列成格狀列表的轉場動畫（進入「全部」列表時的招牌畫面）。

實作備註：Svelte 原生 transition + FLIP 可處理 2/4；snap-scroll 用 CSS scroll-snap 或 IntersectionObserver 驅動；358 張卡同時做飛入動畫需要分批（stagger + 只動 viewport 內的卡）避免掉幀。

## 設計原則（使用者定調）
- 整體操作以直覺為主，不堆功能；每加一個維度都用「徽章/標籤」方式疊加，不增加主分類複雜度。
